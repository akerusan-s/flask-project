Пояснительная записка для проекта "AkraMarket"
Иващенко Сергей

Проект представляет из себя веб-площадку, на которой можно просматривать товары (услуги), добавляемые магазинами, которые создаются пользователями
Пользователи смогут найти нужный им товар (или услугу), а затем просмотреть контактные данные продавца (исполнителя)\

1. Незарегистрированный пользователь
Когда пользователь заходит первый раз на сайт, его перенаправляет на страницу с товарами. Так же оттуда можно перейти на страницу с магазинами. Он может зарег-ться или войти в существующий аккаунт по ссылке в шапке страниц. Пока он этого не сделает он сможет только просматривать товары и магазины (контакты магазина будут показаны только при согласии создателя магазина).

2. Зарегистрированный пользователь (возможности незарегистр. пользователя +)
При регистрации необходимо указать свой email, имя, фамилию, будущий пароль. Для зарегистрированного пользователя становятся доступны следующие возможности:
	- просматривать настройки своего профиля и изменять их
	- создавать свои магазины и изменять их, удалять
	- добавлять товары и магазины в избранное, чтобы быстрее их найти

3. Зарегистрированный пользователь с созданным магазином (возможности зарегистр. пользователя +)
	- такой пользователь сможет добавлять товары, изменять их, добавлять фото товара до 10 штук на товар, удалять
	- такой пользоваетль сможет скрывать товары, принадлежащие его магазинам (не удалять)
	- такой пользователь не сможет добавить в избранное свои товары

API
Реализованы функции get/put/delete/post запросов к сущности "User", get запросов к "Shop"(магазин), "Good"(товар)

1) '/api/v2/users/<int:user_id>/<password>'
user_id - id нужного пользователя, password - пароль к аккаунту пользователю. Если password не верен, то GET вернё краткую информацию о пользователе, иначе полную. Удалять, изменять (DELETE, PUT) можно только при верно введенном пароле.
Параметры для отправки PUT - "name", "email", "surname", "password" (очевидные ключи- один из них, или все сразу (для изменения)
2) '/api/v2/users' - GET вернёт всех пользователей (краткая информация), POST - создаст нового, параметры - "name", "email", "surname", "password" (очевидные ключи).
3) '/api/v2/shops/<int:shop_id>' - get вернёт информацию о магазине по его id
4) '/api/v2/shops' - get вернёт информацию о всех магазинах
5) '/api/v2/goods/<int:good_id>' - get вернёт информацию о товаре по его id
6) '/api/v2/goods' - get вернёт информацию о всех товарах

Примечательные особенности:
	- магазинам можно ставить обложку (фото) и менять её
	- к товару при создании можно добавить лишь 1 фото, но затем добавить ещё до 10 включительно
	- при создании магазина почти невозможно ввести неправильный телефон (строгие проверки)
	- все фотографии при загрузке на сервер фильтруются и изменяются, допустимые расширения - 'png', 'jpg', 'jpeg', 'gif'
